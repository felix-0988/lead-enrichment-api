API Documentation

The Lead Enrichment API provides email and domain enrichment capabilities, delivering detailed company data for any valid email address or domain.

Base URL: https://leads.zenithstudio.app

Authentication

All API requests require an API key passed in the X-API-Key header. 

curl -X POST https://leads.zenithstudio.app/api/enrich/email \
  -H "X-API-Key: le_your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"email": "john@example.com"}'

Endpoints

source string Data source: "hunter", "clearbit", "mock", or "auto"

Response Format

All API responses follow a consistent JSON structure: 

{
  "success": true | false,
  "data": { ... },           // Response data (on success)
  "error": "...",          // Error code (on failure)
  "message": "..."        // Human-readable message
}

Error Handling

The API uses standard HTTP status codes and returns detailed error messages.

Status Error Description
400 Bad Request Invalid request parameters
401 Unauthorized Missing or invalid API key
429 Too Many Requests Rate limit exceeded
500 Internal Server Error Something went wrong on our end

Rate Limits

API requests are limited to 100 requests per 15 minutes per API key on the free plan. Higher limits available on paid plans.

Rate Limit Headers

Header Description
X-RateLimit-Limit Maximum requests allowed per window
X-RateLimit-Remaining Requests remaining in current window
X-RateLimit-Reset Unix timestamp when limit resets
